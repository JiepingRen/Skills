# 销售数据图表生成指南

## 图表设计原则

1. **从宏观到微观**：KPI 概览 → 趋势分析 → 分类对比 → 明细下钻
2. **提供对比基准**：所有数字必须有参照（同比、环比、目标值）
3. **一图一重点**：每个图表聚焦一个分析维度

## 标准图表清单

| # | 图表 | 类型 | 分析目的 |
|---|------|------|----------|
| 1 | 核心KPI卡片 | kpi_card | 一眼看到关键数字 |
| 2 | 销售额趋势 | line | 发现增长/下降趋势 |
| 3 | 区域业绩对比 | bar | 横向比较各团队/区域表现 |
| 4 | 产品销售占比 | pie | 了解产品线结构 |
| 5 | 销售转化漏斗 | funnel | 发现转化瓶颈 |
| 6 | 渠道贡献占比 | doughnut | 评估渠道效率 |
| 7 | 销售排名 Top10 | horizontal_bar | 激励团队，识别标杆 |
| 8 | Top10 客户明细 | table | 关注大客户动态 |

## 数据格式规范

### API 返回格式

每个图表通过 `chart_id` 请求数据，API 需返回对应格式：

**KPI 卡片**:
```json
{
  "total_revenue": 1250000,
  "total_revenue_compare": "+12.5%",
  "revenue_growth": 12.5,
  "quota_completion": 87.3,
  "new_customers": 23,
  "avg_deal_size": 54348
}
```

**折线图/柱状图**:
```json
{
  "labels": ["1月", "2月", "3月"],
  "本期销售额": [100, 120, 150],
  "上期销售额": [90, 110, 130],
  "目标值": [110, 115, 140]
}
```

**饼图/环形图**:
```json
{
  "labels": ["产品A", "产品B", "产品C"],
  "values": [450000, 350000, 200000]
}
```

**漏斗图**:
```json
{
  "stages": ["线索", "意向客户", "商机", "报价", "赢单"],
  "values": [1000, 600, 300, 150, 80]
}
```

**水平柱状图**:
```json
{
  "labels": ["张三", "李四", "王五"],
  "values": [250000, 220000, 180000]
}
```

**表格**:
```json
{
  "rows": [
    ["1", "客户A", "500,000", "12", "41,667", "+15%"],
    ["2", "客户B", "380,000", "8", "47,500", "+5%"]
  ]
}
```

### 本地数据文件

使用 `--data-file` 时，JSON 文件结构为 chart_id → data 的映射：

```json
{
  "kpi_cards": { ... },
  "revenue_trend": { ... },
  "region_comparison": { ... }
}
```

## 输出格式选择

| 格式 | 适用场景 |
|------|----------|
| `png` | 单独使用每张图表，嵌入 PPT/文档 |
| `html` | 完整报告页面，可直接浏览器打开 |
| `json` | 导出原始数据，供其他系统消费 |
