# 销售周报生成指南

## Prompt 设计原则

1. **数据驱动**：以拜访记录为核心数据源，确保周报内容有据可查
2. **汇总去重**：跨多条拜访记录的客户反馈、竞品信息需合并去重
3. **承上启下**：下周计划由本周拜访的"后续计划"自然推导

## 从拜访记录到周报的映射

| 拜访记录字段 | 周报字段 | 映射逻辑 |
|-------------|---------|----------|
| 客户名称 + 拜访时间 | 本周拜访概览 | 统计去重，生成拜访列表 |
| 沟通要点 | 重点进展 | 按客户分组，提炼关键推进 |
| 客户反馈 | 客户反馈汇总 | 跨记录汇总，标注客户来源 |
| 竞品信息 | 竞品动态 | 汇总去重 |
| 后续计划 | 下周计划 | 按时间排序，合并同客户计划 |

## 无拜访记录时的处理

- 如果周期内无拜访记录，依赖补充输入生成周报
- 补充输入也为空时，输出模板框架并全部标注 `[待补充]`

## 质量检查

- [ ] 拜访次数与实际记录数一致
- [ ] 每个拜访客户都在概览表中出现
- [ ] 下周计划可追溯到本周拜访的后续计划
- [ ] 未出现拜访记录和补充输入中不存在的信息
