# 拜访记录生成指南

## Prompt 设计原则

1. **信息提取优先**：从口语化、碎片化输入中提取结构化信息
2. **来源感知**：不同来源的处理策略不同
3. **缺失标注**：无法推断的字段标注 `[待补充]`，不编造
4. **正式化表达**：口语 → 书面语

## 不同输入源处理建议

| 来源 | 特点 | 处理重点 |
|------|------|----------|
| 微信消息 (wechat) | 口语化、碎片化、可能含表情 | 去除口语词和表情，合并碎片信息，提炼要点 |
| 邮件 (email) | 较正式、含格式化内容 | 提取正文核心信息，忽略签名、寒暄 |
| 备忘录 (note) | 简短关键词、速记 | 扩展关键词为完整描述 |
| 通话记录 (call) | 可能是转写文本、口语化 | 类似微信处理，关注承诺和约定 |

## 字段提取策略

| 字段 | 提取线索 |
|------|----------|
| 客户名称 | 公司名、品牌名、"XX那边"、"@XX" |
| 拜访时间 | 日期、"今天"、"上午"、"周X" |
| 拜访人 | 发送者、"我"、签名 |
| 客户对接人 | 人名 + 职位、"X总"、"X经理" |
| 拜访目的 | "聊了一下"、"关于"、邮件主题 |
| 沟通要点 | 讨论的具体内容、数字、方案 |
| 客户反馈 | "他们觉得"、"客户说"、态度词 |
| 竞品信息 | 其他品牌名、"他们也在看"、"对比" |
| 后续计划 | "下周"、"回去之后"、"安排一下" |

## 质量检查

生成后应确认：
- [ ] 必填字段是否都有内容（或标注 `[待补充]`）
- [ ] 沟通要点是否涵盖所有输入中的关键信息
- [ ] 后续计划是否具体可执行
- [ ] 未出现输入中不存在的信息
- [ ] 语言正式、专业
