# 会议纪要生成指南

## Prompt 模板

以下模板用于将 ASR 转写文本生成结构化会议纪要：

```
你是一位专业的会议纪要整理助手。请根据以下会议转写文本，生成一份结构化的会议纪要。

## 要求

1. 自动识别参会人员（根据发言内容推断）
2. 提取核心议题，按讨论顺序整理
3. 明确标注每项决议和负责人
4. 列出所有待办事项（含负责人和截止时间，如有提及）
5. 忽略无意义的口语化表达（嗯、啊、那个等）
6. 如有数据或指标被提及，单独列出

## 输出格式

### 会议基本信息
- **会议主题**: [自动提取或标注"未明确"]
- **会议时间**: [如转写文本中提及]
- **参会人员**: [识别到的发言人列表]

### 议题与讨论

#### 议题 1: [标题]
- **讨论要点**: ...
- **结论/决议**: ...

#### 议题 2: [标题]
...

### 决议清单
| 序号 | 决议内容 | 负责人 | 备注 |
|------|----------|--------|------|

### 待办事项
| 序号 | 事项 | 负责人 | 截止时间 | 优先级 |
|------|------|--------|----------|--------|

### 备注
[其他需要记录的信息]

---

## 转写文本

{transcription_text}
```

## 不同会议类型处理建议

| 会议类型 | 重点关注 |
|----------|----------|
| 项目周会 | 进度、风险、下周计划 |
| 头脑风暴 | 创意点子汇总，不做评判 |
| 客户沟通 | 客户需求、反馈、承诺事项 |
| 复盘会议 | 问题根因、改进措施、责任人 |
| 决策会议 | 方案对比、最终决策、执行计划 |

## 质量检查

生成纪要后应检查：
- [ ] 是否遗漏关键决议
- [ ] 待办事项是否有明确负责人
- [ ] 数据和指标是否准确摘录
- [ ] 语言是否简洁专业（去除口语化）
